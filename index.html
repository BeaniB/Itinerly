▸ First-time setup guide

1. Create the app repo (public)
On github.com, click New repository. Name it e.g. road-trip-planner. Keep it Public. Create it.

2. Upload the app files
In the new repo click Add file → Create new file. Paste the contents of index.html, sw.js, manifest.json, and icon.svg one by one.

3. Enable GitHub Pages
Go to Settings → Pages → Source: Deploy from branch → main / root → Save. Your app will be live at https://USERNAME.github.io/road-trip-planner/ within ~60 s.

4. Create the private data repo
Create a second repo, e.g. road-trip-data. Set it to Private. No files needed — the app creates trips.json on first sync.

5. Generate a Personal Access Token
Go to github.com/settings/tokens → Generate new token (classic). Check the ☑ repo scope. Copy the token (shown only once).

6. Configure sync here
Paste your token above, enter your GitHub username and the data repo name, then click Save & close. Hit the ☁ button in the toolbar to sync.
